## 附录：删除已发布的 npm 包或版本

### 删除特定版本

```bash
# 删除指定版本
npm unpublish dev-proxy-cookie@<version>

# 示例：删除 beta.5 版本
npm unpublish dev-proxy-cookie@1.0.1-beta.5
```

### 删除整个包（谨慎操作）

```bash
# 删除整个包（需要确认）
npm unpublish dev-proxy-cookie --force
```

### 重要注意事项

| 限制条件             | 说明                                                          |
| -------------------- | ------------------------------------------------------------- |
| **时间限制**   | 发布后 24 小时内可以删除版本，超过 24 小时需要联系 npm 支持   |
| **依赖警告**   | 如果有其他包依赖该版本，删除会影响其他用户                    |
| **latest tag** | 删除 latest tag 的版本会导致用户无法通过 `npm install` 安装 |
| **强制删除**   | 使用 `--force` 参数可以强制删除，但不推荐                   |

### 删除后的恢复

如果误删了版本，可以重新发布相同版本（如果还在 24 小时内）：

```bash
# 重新发布相同版本
npm publish --tag <tag>
```

### 替代方案：废弃版本

如果版本发布时间超过 24 小时或不想完全删除，可以标记为废弃：

```bash
# 废弃指定版本
npm deprecate dev-proxy-cookie@1.0.0 "该版本已废弃，请升级到最新版本"

# 废弃所有版本
npm deprecate dev-proxy-cookie "该包已废弃，请使用替代方案"
```

### 常见问题

**Q: 删除版本时提示权限不足？**

A: 确保使用的 npm 账号是包的所有者或具有删除权限：

```bash
# 检查当前登录账号
npm whoami

# 如果不是正确账号，重新登录
npm login
```

**Q: 删除后其他用户会怎样？**

A: 已安装该版本的用户不受影响，但新用户无法安装被删除的版本。

**Q: 如何查看已发布的所有版本？**

A:

```bash
npm view dev-proxy-cookie versions
```
